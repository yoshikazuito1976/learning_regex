# 📝 `grep -E` 正規表現 練習問題

このファイルは、`grep -E`（拡張正規表現）を使った検索の練習問題集です。
対象のファイル名は `regex.txt` と想定して、コマンドを考えてみましょう！

答えは「👉 答えと解説を見る」をクリックすると展開されます。

---

## 第1問：行末の指定とエスケープ
**問題:**
テキストの中から、行の最後が `.`（ピリオド）で終わっている行だけを検索するコマンドを書いてください。

<details>
<summary>👉 答えと解説を見る</summary>

```bash
grep -E '\.$' regex.txt

```
【解説】

\. : ピリオド . は「任意の1文字」を表す特別な記号（メタ文字）なので、その直前にバックスラッシュ \ を置いて「ただのピリオド」として認識させます（エスケープ）。

$ : 行の末尾を表すアンカーです
</details>

## 第2問：文字クラスと回数指定
問題:
テキストの中から、「ちょうど3桁の数字」（例: 123, 999, 045 など）が含まれる行を検索するコマンドを書いてください。

<details>
<summary>👉 答えと解説を見る</summary>

```Bash
grep -E '[0-9]{3}' regex.txt
```
【解説】

[0-9] : 0から9までの「いずれか1文字の数字」を表す文字クラスです。

{3} : 直前のパターン（この場合は数字）が「ちょうど3回繰り返される」ことを指定します。
</details>
---

## 第3問：論理和 (OR)
問題:
テキストの中から、「start」または「stop」のどちらかの単語が含まれている行を検索するコマンドを書いてください。

<details>
<summary>👉 答えと解説を見る</summary>

```bash
grep -E 'start|stop' regex.txt
```

【解説】


| : パイプ記号は拡張正規表現で「OR（または）」を意味します。左側か右側のどちらかにマッチすればOKとなります。

</details>

---

## 第4問：否定の文字クラス
問題:
テキストの中から、「数字以外」の文字で始まっている行を検索するコマンドを書いてください。（アルファベットや記号などで始まる行）

<details>
<summary>👉 答えと解説を見る</summary>

```bash
grep -E '^[^0-9]' regex.txt
```

【解説】

^ (外側) : 行の先頭を表すアンカーです。

[^0-9] : [] の中にある ^ は「否定」を意味します。つまり、「0〜9の数字以外の任意の1文字」という意味になります。

</details>

## 第5問：グループ化

問題:
テキストの中から、「go」という文字列が2回以上連続している部分（例: gogo, gogogo）を検索するコマンドを書いてください。

<details>
<summary>👉 答えと解説を見る</summary>

```bash
grep -E '(go){2,}' regex.txt
```

【解説】

(go) : go という2文字をひとつのグループとしてまとめます。

{2,} : 直前のパターン（ここでは go というグループ）が「2回以上」繰り返されることを指定します（カンマの後に数字を書かないと「〜以上」という意味になります）。

</details>

## 第6問：部分一致の罠と「単語の境界」

**問題:**
テキストの中から、「ちょうど3桁の数字」だけを単語として抽出したいとします。（例: `404` や `205` にはマッチするが、`2024` や `xyz123` にはマッチさせない）
どのようなコマンドを書けばよいでしょうか？

<details>
<summary>👉 答えと解説を見る</summary>

```bash
grep -E '\b[0-9]{3}\b' regex.txt
```
【解説】

もし [0-9]{3} だけだと、2024 の中の 202 や、xyz123 の中の 123 など、「3桁以上連続している数字の一部」にもマッチしてしまいます（部分一致の罠）。

\b は 「単語の境界 (Word Boundary)」 を表します。

\b[0-9]{3}\b と両端を囲むことで、「前後に他の文字（アルファベットや数字）がくっついていない、独立した3桁の数字」だけを正確に狙い撃ちできます！

※注：xyz123 はアルファベットと数字がくっついていて「1つの単語」とみなされるため、123 の前に境界線(\b)は存在せず、マッチしません。

</details>
